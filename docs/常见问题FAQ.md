# ❓ 常见问题FAQ

## 📚 关于仓库结构

### Q1: dushu仓库和sanxiangwendao仓库是什么关系？

**答**：
- **dushu仓库**（本仓库）：读书系统本身
  - 系统指令
  - 笔记模板
  - 书单
  - 使用文档
  
- **sanxiangwendao仓库**：知识库来源
  - 78篇理论文档
  - 商业案例
  - 杨国平思想体系
  
**关系**：dushu系统会**自动关联**sanxiangwendao仓库的理论，形成"书籍理论 + 仓库理论"的融合。

---

### Q2: 我需要下载sanxiangwendao仓库吗？

**答**：取决于你如何使用。

**场景A：只用Claude Projects**
- 不需要下载到本地
- 在Claude Projects的Project Knowledge中上传sanxiangwendao的核心文档即可
- AI会自动搜索和关联

**场景B：本地查阅 + Claude Projects**
- 建议下载到本地（如 `E:\sanxiangwendao\`）
- 方便本地查阅文档
- 同时在Claude Projects中上传

**下载方式**：
```bash
git clone https://github.com/MRYGP/sanxiangwendao.git
```

---

### Q3: 我应该把笔记放在哪个仓库？

**答**：
- **读书笔记** → 放在 `dushu` 仓库的 `reading-notes/`
- **智慧卡片** → 放在 `dushu` 仓库的 `knowledge-cards/`
- **行动日志** → 放在 `dushu` 仓库的 `action-logs/`
- **理论文档** → 保持在 `sanxiangwendao` 仓库（不要移动）

**原则**：dushu是你的个人阅读系统，sanxiangwendao是公共知识库。

## 🤖 关于Claude Projects

### Q4: Claude Projects是什么？必须用吗？

**答**：
- **Claude Projects** = Claude的项目功能，支持Custom Instructions + Project Knowledge
- **是否必须**：不是，但强烈推荐
  - ✅ 用Claude Projects：自动搜索仓库理论，体验最佳
  - ⚠️ 不用Claude Projects：需要手动提供理论文档，体验较差

**替代方案**：
- 用普通的Claude对话 + 手动粘贴理论文档
- 用DEEPSEEK + 手动提供文档
- 用本地AI（如Ollama）+ RAG系统

---

### Q5: Project Knowledge上传了很多文档，AI会自动搜索吗？

**答**：
- ✅ **会自动搜索**，这是Claude Projects的核心功能
- 当你输入"我想读《从0到1》"时，AI会：
  1. 识别主题（如"垄断"、"创新"）
  2. 自动搜索Project Knowledge中的相关文档
  3. 在对话中展示"🔗 仓库理论映射"

**如果没有自动搜索**：
- 检查Project Knowledge是否成功上传
- 检查Custom Instructions是否正确保存
- 手动触发："请搜索仓库中关于'护城河'的文档"

---

### Q6: 我上传了100篇文档，AI会不会混乱？

**答**：
- ✅ **不会混乱**，Claude的语义搜索很强大
- AI只会搜索与当前主题相关的文档
- 建议优先上传核心20-30篇文档，按需添加其他文档

**推荐策略**：
- **P0核心文档**（20篇）：必须上传
- **P1重要文档**（30篇）：推荐上传
- **P2补充文档**（28篇）：按需上传

详见：[配置指南](../claude-projects/配置指南.md)

## 📖 关于阅读流程

### Q7: 我应该多快读完一本书？

**答**：
- **理论类书籍**（如《从0到1》）：1-2周
- **案例类书籍**（如《巴菲特致股东信》）：1周
- **传记类书籍**（如《硅谷钢铁侠》）：1-2周

**重点**：不是速度，而是深度。
- ✅ 慢一点，但真正理解和应用
- ❌ 快速浏览，但什么都没记住

**推荐节奏**：
- 每天1-2个对话单元（30-60分钟）
- 每周完成1本书
- 每月完成4-5本书

---

### Q8: 我没有时间写这么详细的笔记怎么办？

**答**：根据时间选择合适的版本。

**最小化版本**（每本书30分钟）：
- 只提炼智慧卡片（3-5张）
- 跳过练习和行动日志

**标准版本**（每本书2-3小时）：
- 笔记 + 智慧卡片 + 练习
- 跳过行动日志

**完整版本**（每本书5-7天）：
- 笔记 + 智慧卡片 + 练习 + 行动日志
- 持续实践和反思

**建议**：先用标准版本，体验完整流程后再调整。

## 🎯 关于实战练习

### Q10: 练习题太难了，我答不出来怎么办？

**答**：
- ✅ **这是正常的**，练习题设计就是要让你深度思考
- 建议：
  1. 先尝试写出你能想到的（哪怕只有200字）
  2. 提交答案，查看AI的标准答案
  3. 对比差距，找出盲区
  4. 下次练习时改进

**重点**：练习的目的不是"答对"，而是"发现盲区"。

---

### Q11: 我的练习得分只有60分，是不是太低了？

**答**：
- ✅ **60分已经很好了**，说明你理解了核心概念
- 得分分布：
  - 40-60分：理解核心，但深度不够
  - 60-75分：理解深入，但融合不够
  - 75-85分：融合较好，但有细节问题
  - 85-100分：优秀，深度洞察

**建议**：
- 不要追求满分，追求进步
- 对比每次练习的得分，看到提升
- 重点关注AI给出的"深度洞察"

## 🔗 关于理论融合

### Q12: "书中理论"和"仓库理论"有冲突怎么办？

**答**：
- ✅ **冲突是好事**，说明你在深度思考
- 处理方式：
  1. 理解冲突的本质（不同视角？不同场景？）
  2. 分析两者的适用边界
  3. 形成自己的融合洞察

**示例**：
- 巴菲特的"护城河"（被动防御）
- vs 仓库理论"规则制定者"（主动进攻）
- 融合：初期用"规则制定"打破旧秩序，中期用"护城河"守住优势，长期再次"规则制定"

## 🛠️ 技术问题

### Q14: 我应该用Git管理笔记吗？

**答**：
- ✅ **强烈推荐**，Git可以：
  - 版本管理（看到每次修改）
  - 多设备同步
  - 备份（防止丢失）
  
**基本Git操作**：
```bash
# 初始化
cd E:\dushu
git init
git add .
git commit -m "feat: 初始化读书系统"

# 连接GitHub
git remote add origin https://github.com/你的用户名/dushu.git
git push -u origin main

# 日常使用
git add .
git commit -m "docs: 添加《从0到1》笔记"
git push
```

## 💡 关于持续学习

### Q17: 如何保持长期学习的动力？

**答**：

**策略1：设定明确目标**
- 短期目标：3个月读10本书
- 中期目标：1年读40本书
- 长期目标：3年读100本书，形成认知体系

**策略2：加入读书小组**
- 找2-3个朋友一起读
- 每周分享学习心得
- 互相监督和鼓励

**策略3：看到进步**
- 每月回顾，统计阅读数量
- 对比练习得分的变化
- 记录应用理论后的实际效果

**策略4：获得正反馈**
- 将学习成果应用到工作
- 看到决策质量的提升
- 获得同事/领导的认可
