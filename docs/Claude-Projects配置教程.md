# 🎓 Claude Projects配置教程

本教程详细说明如何配置Claude Projects，实现"书籍理论 + 仓库理论"的自动融合。

---

## 📋 前提条件

- ✅ Claude付费账号（Pro或Team版本）
- ✅ 已有GitHub账号
- ✅ 已clone本仓库到本地（`E:\dushu\`）
- ✅ 了解sanxiangwendao仓库的位置

---

## 🚀 配置流程

### Step 1: 创建Claude Projects

#### 1.1 访问Projects页面

打开浏览器，访问：
```
https://claude.ai/projects
```

#### 1.2 创建新项目

1. 点击右上角的 **"New Project"** 按钮
2. 在弹出的对话框中：
   - **Project Name**：输入 `思考，推理，读书`
   - （其他选项保持默认）
3. 点击 **"Create Project"**

✅ **验证**：你应该看到一个新的空白项目页面。

---

### Step 2: 配置Custom Instructions

#### 2.1 打开Project Settings

1. 在项目页面，点击右上角的 **齿轮图标** 或 **"Project Settings"**
2. 在左侧菜单中，找到 **"Custom Instructions"**

#### 2.2 复制系统指令

1. 在本地打开文件：
   ```
   E:\dushu\claude-projects\深度阅读系统v3.0-完整指令.md
   ```

2. **全选并复制**（Ctrl+A → Ctrl+C）
   - 从第一行开始
   - 到最后一行结束
   - 确保没有遗漏

#### 2.3 粘贴到Custom Instructions

1. 在Custom Instructions的文本框中，**粘贴**复制的内容
2. 检查内容是否完整：
   - [ ] 开头：`深度阅读与认知融合系统 v3.0`
   - [ ] 包含：`你将扮演我的深度阅读与认知融合导师`
   - [ ] 包含：`强制仓库搜索机制`
   - [ ] 包含：`Phase 0-5`的完整流程
   - [ ] 结尾：完整的52本书单

3. 点击 **"Save"** 按钮

✅ **验证**：保存成功后，刷新页面，检查Custom Instructions是否还在。

⚠️ **常见问题**：
- **如果提示"超过字符限制"**：
  - Custom Instructions有约50000字符限制
  - 解决方案：删除指令中的详细示例部分（保留核心流程）
  
---

### Step 3: 添加Project Knowledge

这是**最关键的一步**，决定了AI能否自动关联仓库理论。

#### 3.1 理解Project Knowledge

**Project Knowledge** = Claude Projects的知识库功能
- 你上传的文档会被AI自动索引
- AI会根据对话主题，自动搜索相关文档
- 搜索结果会融入AI的回答中

**关键**：
- ✅ 上传的文档必须是 `.md` 或 `.txt` 格式
- ✅ 文档内容要清晰、结构化
- ✅ 建议先上传核心20-30篇，再按需添加

#### 3.2 准备上传文档

```
术层方法论（10篇）：
```

#### 3.3 上传文档

**方式A：手动上传（推荐）**

1. 在Project Settings中，找到 **"Project Knowledge"**
2. 点击 **"Add files"** 按钮
3. 从本地选择文件：
   - 如果你已clone了sanxiangwendao仓库：
     ```
     E:\sanxiangwendao\01-dao\framework\思想体系核心框架 v1.1.md
     E:\sanxiangwendao\02-shu\innovation\创新三元法.md
     ...
     ```
   - 如果没有clone，先执行：
     ```bash
     cd E:\
     git clone https://github.com/MRYGP/sanxiangwendao.git
     ```

4. **分批上传**（建议每批10-15个文件）：
   - 第1批：道层理论（4篇）
   - 第2批：术层方法论（10篇）
   - 第3批：商业案例（3篇）
   - 第4批：杨国平思想体系（3篇）

5. 等待上传完成：
   - 每个文件上传后，会显示在文件列表中
   - 确认所有文件都上传成功

**方式B：连接GitHub仓库（如果支持）**

⚠️ **注意**：Claude Projects可能不支持直接连接GitHub，以方式A为准。

如果支持：
1. 在Project Settings中，找到 **"Connect GitHub"**
2. 授权Claude访问你的GitHub
3. 选择仓库：`MRYGP/sanxiangwendao`
4. 选择要同步的目录
5. 设置自动同步

#### 3.4 添加本仓库的书单

从dushu仓库上传：

```
E:\dushu\book-lists\创业者书单-50本.md
E:\dushu\book-lists\投资者书单-52本.md
```

这样AI就能知道有哪些书可以读。

- 在Project Knowledge中，应该能看到至少23个文件
- 点击某个文件，确认内容完整

## ⚠️ 故障排查

### 问题1：AI没有自动搜索仓库

**症状**：对话中没有"🔍 正在搜索知识库"

**原因1**：Project Knowledge未正确添加
- **检查**：Project Settings → Project Knowledge，确认文件列表不为空
- **解决**：重新上传文档

**原因2**：Custom Instructions未保存
- **检查**：刷新页面，查看Custom Instructions是否还在
- **解决**：重新粘贴并保存

**原因3**：对话缓存问题
- **检查**：创建新对话，重新测试
- **解决**：点击"New Chat"，重新输入测试命令

---

### 问题2：AI搜索结果不准确

**症状**：AI关联的文档与主题不符

**原因1**：文档内容不匹配
- **检查**：手动查看文档，确认是否包含相关关键词
- **解决**：手动告诉AI："请参考《具体文档名》"

**原因2**：关键词模糊
- **检查**：你输入的书名或主题是否明确
- **解决**：使用更精确的关键词，如"我想学习'第一性原理'"

---

### 问题3：上传文档失败

**症状**：上传卡住或报错

**解决方案**：
1. **检查文件格式**：必须是 `.md` 或 `.txt`
2. **检查文件大小**：单个文件不超过10MB
3. **分批上传**：每批10-15个文件
4. **清除缓存**：清除浏览器缓存后重试
5. **更换浏览器**：尝试Chrome或Edge

---

### 问题4：Custom Instructions超过字符限制

**症状**：保存时提示"超过50000字符"

**解决方案**：

**方案A：删除示例部分**
- 保留核心流程（Phase 0-5）
- 删除详细示例（每个Phase中的示例）
- 保留52本书单（AI需要知道有哪些书）

**方案B：精简书单**
- 只保留你近期会读的20本书
- 其他书在需要时手动添加

**方案C：分拆为多个项目**
- 创建多个专题项目（如"创业阅读"、"投资阅读"）
- 每个项目独立的Custom Instructions
