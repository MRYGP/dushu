# 📖 读书系统Claude Projects完整配置指南

## 一、创建新项目

### 1.1 项目基本信息
- **项目名称**：思考，推理，读书（或"深度阅读系统"）

## 五、常见问题与解决方案

### Q1: AI没有自动搜索仓库

**症状**：对话中没有"🔍 正在搜索知识库"

**原因**：
- Project Knowledge未正确添加
- Custom Instructions未保存

**解决**：
1. 检查Project Settings → Project Knowledge
2. 确认文档列表不为空
3. 重新保存Custom Instructions
4. 重启对话（新建对话）

### Q2: 搜索结果不准确

**症状**：搜索"护城河"返回无关文档

**原因**：
- 文档内容与搜索关键词不匹配
- 文档格式问题

**解决**：
1. 手动检查文档内容，确认是否包含关键词
2. 使用更精确的关键词
3. 手动告诉AI："请参考《规则制定者.md》"

### Q3: Custom Instructions字符数超限

**症状**：保存时提示"超过50000字符"

**解决方案A**：精简指令
- 删除详细示例
- 保留核心流程

**解决方案B**：分拆项目
- 创建多个专题项目
- 每个项目独立指令

### Q4: 上传文档失败

**症状**：上传卡住或报错

**解决**：
- 分批上传（每批10个文件）
- 检查文件大小（单个文件<10MB）
- 检查文件格式（必须是.md）
- 清除浏览器缓存后重试

## 六、推荐工作流

### 场景1：系统性阅读一本书

```
Day 1: 选择书籍 → Phase 1配置焦点
Day 2-3: Phase 2深度对话（每天1-2个单元）
Day 4: Phase 3实战练习
Day 5: Phase 4理解检查
Day 6: Phase 5总结 + 制定行动计划
Day 7: 休息，让知识沉淀
```

### 场景2：快速提取核心智慧

```
1. 直接选择"核心理念筑基"模块
2. 重点学习2-3个核心概念
3. 跳过练习，直接总结
4. 时间：1-2小时
```

### 场景3：对比阅读（多本书）

```
1. 先读书A（如《从0到1》）
2. 完成后，立即读书B（如《精益创业》）
3. 在Phase 5总结时，让AI对比两本书的异同
4. 提炼跨书籍的深层洞察
```

## 八、高级功能

### 8.1 多项目配置

如果你需要不同的阅读场景，可以创建多个项目：

- **项目1：创业阅读**（侧重创新方法论）
- **项目2：投资阅读**（侧重财务思维）
- **项目3：管理阅读**（侧重团队与文化）

每个项目：
- 独立的Custom Instructions
- 不同的Project Knowledge（精选相关文档）
- 专注的书单

### 8.2 与其他工具集成

- **Cursor**：本地编辑笔记和文档
- **GitHub**：版本管理和同步
- **Notion**：结构化整理智慧卡片
- **Obsidian**：构建知识图谱
